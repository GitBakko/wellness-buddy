<objective>
First-prod-deploy on 2026-05-04 surfaced parser format mismatch. Real plans (Stefano + Marta) use weekly grid `| Giorno | Opzione A | Opzione B |` for lunches and dinners, NOT `### Opzione X` subheadings. Plan 02-02's weekly variant model assumed week-level A/B/Speciale; reality requires per-day variants. This plan refactors parser, schema, and frontend to handle grid format with per-day variant selection while preserving subheading-format backward compat for EXAMPLE.md fixtures.

Purpose: Gap-closure between dev fixture format and real-world plan format. Without this plan, /today and /settimana cannot consume the actual MD files Stefano + Marta authored. Plans 02-05 (shopping), 02-06 (PDF), and 02-07 (family sync) all depend on grid-aware lunches/dinners structure to extract ingredients and aggregate variants per-day.

Output: Backend parser + schema + migration + service layer support per-day variants from weekly grid; frontend /settimana renders 7-day grid with lunch + dinner variant selectors per row; /today shows the day's specific recipe title (not generic "Pranzo Opzione A"); both real plans parse cleanly end-to-end.
</objective>
