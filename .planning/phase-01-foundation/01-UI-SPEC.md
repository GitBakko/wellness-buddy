## 1. Design System

| Property | Value | Rationale |
|----------|-------|-----------|
| Tool | **shadcn/ui CLI v4** + custom Tailwind 4 `@theme` layer | Copy-paste ownership over components, full token control. PROJECT.md `@theme` constraint. Vanilla shadcn defaults are explicitly **rejected** by UI-03 — every component is customized after `npx shadcn add`. |
| Preset | **none — custom CSS-first via `@theme`** | shadcn 2026 supports `@theme` natively; we author tokens directly in `app.css`, do not use a registry preset. `components.json` to be initialized in Plan 01 (Frontend skeleton). |
| Component library | **shadcn/ui (Radix UI primitives)** | WAI-ARIA built-in. Sole component source for Phase 1. Headless UI / Material UI explicitly rejected (PITFALLS, STACK §"What NOT to Use"). |
| Icon library | **lucide-react** (single source) | UI-19 + STACK §iconography. Stroke 1.75 default (slightly heavier than 1.5 = friendlier without losing minimal feel). Sizes 16 / 20 / 24 / 32 only. **Never mix with emoji in UI chrome** — emoji allowed only in copy strings, ≤1-2 per screen (UI-19). |
| Font — UI/Body | **Geist Sans** (npm `geist`) | STACK + Win Requisite. Replaces Inter (rejected as "neutral SaaS"). Variable font, `font-feature-settings: 'cv11','ss01'` enabled. |
| Font — Numerals | **Geist Mono** with `font-variant-numeric: tabular-nums` | KPI cards, weight values, macro display, time. Stops digit-jitter on live updates. |
| Font — Display (optional) | **Instrument Serif** | Reserved for `/today` page hero greeting only ("Buongiorno, Stefano"), max one heading per page. Adds editorial elegance without infantilizing. Loaded `font-display: swap`. |
| Illustration system | **Storyset** (primary) + **Open Doodles** (spot accents) | Both colorize-able to brand tokens. unDraw monochrome blue **rejected** (FEATURES anti-pattern). |
| Animation | **Motion v12** (`motion/react`) + **tailwindcss-animate** for shadcn defaults | Spring physics, never linear. Lottie reserved for Phase 3 milestones. **No Rive in Phase 1** (Phase 3 only). |
| Toasts | **sonner** (single import) | shadcn default. Custom-styled to brand palette. |

**Rejected patterns (codified for Phase 1):**
- Vanilla shadcn (no customization)
- Inter / Roboto / system-ui as primary
- Material UI / Mantine / Chakra / Bootstrap
- Animated character coach, sound effects, treasure-chest reveals
- Cartoon mascot in chrome (mascot Phase 3 only, milestones+empty states only)

## 2. Spacing Scale (Tailwind 4 `@theme` — multiples of 4)

```css
@theme {
  --spacing-0: 0;
  --spacing-px: 1px;        /* hairline borders only */
  --spacing-1: 4px;         /* xs — icon-to-text gap */
  --spacing-2: 8px;         /* sm — compact internal padding */
  --spacing-3: 12px;        /* between md and sm — exception (form row gutter) */
  --spacing-4: 16px;        /* md — DEFAULT card/element padding */
  --spacing-5: 20px;        /* exception — top of /today scroll list */
  --spacing-6: 24px;        /* lg — section padding */
  --spacing-8: 32px;        /* xl — layout gaps */
  --spacing-10: 40px;       /* exception — bottom-tab safe rest area */
  --spacing-12: 48px;       /* 2xl — major section breaks */
  --spacing-16: 64px;       /* 3xl — page top margin desktop */
  --spacing-20: 80px;       /* hero vertical breathing on /today desktop only */
}
```

| Token | Value | Usage |
|-------|-------|-------|
| `xs` | 4px | Icon ↔ text gap, hairline alignment |
| `sm` | 8px | Compact list rows, badge internal padding |
| `md` | 16px | **Default** card padding, form field gap, paragraph rhythm |
| `lg` | 24px | Card-to-card gap, section padding |
| `xl` | 32px | Layout container padding mobile, dashboard card gap |
| `2xl` | 48px | Major section break (auth screen vertical rhythm) |
| `3xl` | 64px | Desktop page top, between-feature breathing |

**Touch-target exception (UI-06 + iOS HIG):** Any tap target must be ≥ **44 × 44 px** physical hit area regardless of visual size. Icon-only buttons render at 24px icon inside a 44 × 44 invisible hit-area wrapper.

**Safe-area exceptions (PWA on iOS notch):**
- Top app bar: `padding-top: max(--spacing-4, env(safe-area-inset-top))`
- Bottom tab bar: `padding-bottom: max(--spacing-4, env(safe-area-inset-bottom))`
- Sheets / modals from bottom: same `env(safe-area-inset-bottom)` rule

**Container queries (UI-02):** Cards in dashboard grid use `@container` not `@media`. Breakpoints `xs: 320`, `sm: 480`, `md: 768`, `lg: 1024`, `xl: 1280`. **Mobile-first 390px is the design target** — desktop is the adaptation, not the reverse.

## 3. Typography

Three sizes (sm/base/lg) on body axis, four heading sizes. **Two weights** — `regular (400)` and `semibold (600)`. No bold (700) for body, no thin (300) anywhere (illegibility on iOS retina dark mode). Display weight 500 only on Instrument Serif hero.

| Role | Font | Size (px) | Weight | Line Height | Letter Spacing | Usage |
|------|------|-----------|--------|-------------|----------------|-------|
| Caption | Geist Sans | 12 | 400 | 1.4 | 0 | Timestamps, helper text under inputs, "Last synced X min ago" |
| Body sm | Geist Sans | 14 | 400 | 1.5 | 0 | Compact lists, secondary text, table cells |
| Body | Geist Sans | 16 | 400 | 1.5 | 0 | **Default** running text, form labels, paragraph copy |
| Body emphasis | Geist Sans | 16 | 600 | 1.5 | 0 | In-line emphasis, primary button label |
| KPI value | Geist Mono | 28 | 500 | 1.1 | -0.01em | Weight, kcal, streak count — `tabular-nums` |
| KPI value lg | Geist Mono | 36 | 500 | 1.1 | -0.01em | Hero KPI on dashboard (Phase 3 mostly; reserve token now) |
| Heading sm | Geist Sans | 18 | 600 | 1.3 | -0.005em | Card title, modal title |
| Heading | Geist Sans | 22 | 600 | 1.25 | -0.01em | Section heading on `/today` |
| Heading lg | Geist Sans | 28 | 600 | 1.2 | -0.015em | Page title (`/today`, `/settings`) |
| Display | Instrument Serif | 36 | 500 | 1.1 | -0.02em | One-per-page hero greeting only |

```css

  --font-sans: "Geist Sans", system-ui, sans-serif;
  --font-mono: "Geist Mono", ui-monospace, monospace;
  --font-display: "Instrument Serif", "Geist Sans", serif;
  --text-caption: 0.75rem;     /* 12 */
  --text-sm: 0.875rem;         /* 14 */
  --text-base: 1rem;           /* 16 */
  --text-kpi: 1.75rem;         /* 28 */
  --text-kpi-lg: 2.25rem;      /* 36 */
  --text-h-sm: 1.125rem;       /* 18 */
  --text-h: 1.375rem;          /* 22 */
  --text-h-lg: 1.75rem;        /* 28 */
  --text-display: 2.25rem;     /* 36 */
  --leading-tight: 1.1;
  --leading-heading: 1.2;
  --leading-snug: 1.3;
  --leading-base: 1.5;

```

**Numeric formatting (UI-18):** Always render via `Intl.NumberFormat('it-IT')`. Weight `75,3 kg`. Calories `1.250 kcal`. Date short `lun 5 mag`. Time **24h** `19:30`. Sorting via `Intl.Collator('it')` and NFC-normalize on save. NEVER `75.3 kg` or `7:30 PM`.

**Anti-patterns (locked):**
- ❌ Three or more font weights in same view
- ❌ Bold (700) for body — too heavy with Geist
- ❌ Italic for emphasis (use weight)
- ❌ ALL CAPS labels except for one-letter status badges
- ❌ Mixing Geist Sans tabular and proportional in same number column

## 4. Color (OKLCH, dark-mode first-class)

**60/30/10 strategy:**
- **60% surface** — `--color-bg` (page background) + `--color-surface` (cards)
- **30% structure** — neutrals 100–300 (borders, dividers, secondary text)
- **10% accent** — `--color-accent-coral` reserved-for list below

**Color reservation contract (10% accent):**

The **warm-coral** accent is reserved for — and ONLY for:
1. Primary CTA button background (`Button variant="primary"`)
2. Active state on bottom-tab + nav-rail icon
3. Selected variant in segmented control (Phase 2, but token defined now)
4. Linked text inside paragraphs (underline-on-hover only)
5. Streak counter flame glyph (Phase 3, token reserved)

**It is forbidden** for:
- Card borders, dividers, generic interactive borders
- Secondary buttons, ghost buttons
- Icons in routine chrome (use neutral 700 on light, 200 on dark)
- Non-primary form input focus rings (use neutral 800 ring)
- Decorative flourish

**Semantic colors** are reserved for their semantic event only. `success` never used as decorative green; `destructive` never used as a "stop" metaphor outside actual destructive intent.

## 5. Motion budget (UI-04, UI-05, UI-06)

```css
@theme {
  --duration-instant: 80ms;        /* tap scale */
  --duration-fast: 150ms;          /* hover, focus ring fade-in */
  --duration-base: 250ms;          /* default state transition (max for micro) */
  --duration-slow: 400ms;          /* sheet/modal in/out — reserve, rarely used */
  --duration-celebration: 800ms;   /* HARD MAX for Phase 3 Lottie */
  --ease-out-soft: cubic-bezier(0.22, 1, 0.36, 1);
  --ease-spring: cubic-bezier(0.34, 1.56, 0.64, 1);  /* light overshoot for celebrations */
}
```

**Hard rules (enforced by code review + axe-like Playwright check):**
1. Max **250ms** for any micro-transition (state, color, position)
2. Max **800ms** for celebration moments (Phase 3+ — none in Phase 1)
3. Max **2 simultaneous moving elements** per viewport
4. `ease-out-soft` (or spring for celebrations) — **never linear** except for opacity-fade-to-skeleton
5. `prefers-reduced-motion: reduce` → all transforms become instant. Implementation: every `motion.div` reads `useReducedMotion()` from `motion/react` and falls back to no-op variants.
6. **Tap microinteraction:** every Button/IconButton scales to 0.97 over 80ms `ease-out-soft` on press, returns 150ms. Implemented once in shared `Button` component, never per-screen.
7. **Layout shift forbidden** — no entry animation that moves content already on screen >2px.
8. **≤1 celebration per session** (Phase 3 enforcement; Phase 1 has no celebrations to enforce).

**Phase 1 specific motion list (the only animations allowed):**
- Page transition: 250ms opacity-only crossfade (route change)
- Toast in/out: 250ms slide-up + fade via sonner default
- Button tap scale: 80ms instant + 150ms return
- Checkbox tick: SVG path stroke-dasharray draw 200ms
- Form field focus ring: 150ms opacity fade-in
- Sheet/modal: 250ms slide-up from bottom on mobile, 200ms scale+fade on desktop
- Skeleton shimmer: 1.5s linear loop (the only linear easing allowed)
- Update toast appearance: sonner default

**Forbidden in Phase 1:**
- Lottie (Phase 3)
- Rive (Phase 3)
- Confetti (Phase 3, first-weight celebration only)
- Layout animations (`<motion.div layout>`) — defer to when needed

## 7. Copywriting Contract

**Italian-only Sprint 1** — every string lives in `frontend/src/i18n/copy.it.ts` (FND-09). No hardcoded strings in components. Tone: *warm, slightly playful but never silly* (UI-17). NFC normalize on save, sort with `Intl.Collator('it')`.

### 7.1 Tone-of-voice rules (locked)

1. **No `!` in error or status copy.** Reserved for celebratory moments only (Phase 3 — none in Phase 1).
2. **No infantilizing.** No "Ops!", no "Yay!", no "Hai fatto WOW!", no leetspeak, no caps lock.
3. **Specific over generic.** "Sessione scaduta dopo 7 giorni di inattività. Accedi di nuovo." NOT "Sei stato disconnesso."
4. **Verb-first CTAs.** "Carica piano", "Salva peso", "Segna pasto", "Accedi", "Esci".
5. **Italian conventions.** "Lunedì" capital, "kg" lowercase no period, "kcal" lowercase no period, time `19:30`, date `lun 5 mag` short / `lunedì 5 maggio` long.
6. **Emoji budget ≤1-2 per screen, copy only**, never in chrome (UI-19). Phase 1 emoji whitelist: 💪 (workout-related), 🌙 (theme toggle dark), ☀️ (theme toggle light), 🥗 (allowed only on `/today` empty state). Forbidden: 🎉 🎊 🔥 (Phase 3 streak only) ⚡ 🚀 🔔 (notifications use lucide bell icon).
7. **Address user by name when possible.** "Buongiorno, Stefano" not "Buongiorno utente".
8. **Plurals correctly.** "1 pasto" / "2 pasti", "1 minuto" / "10 minuti", "7 giorni di fila" not "7 giorno".
9. **Macro language.** "proteine" lowercase, "carboidrati" or "carbo" abbreviated allowed in chips, "grassi" not "lipidi".

## 10. PWA-Specific Patterns

### 10.1 iOS install (FND-05, Pitfall #UX-1)

- Detect `window.matchMedia('(display-mode: standalone)')` — if false **and** UA matches iOS Safari → show install banner with illustrated step-by-step
- Banner is **dismissible**, dismissal stored 7 days in localStorage with key `wb-install-banner-dismissed-until`
- Banner uses Storyset illustration of iOS share sheet with "Add to Home Screen" highlighted
- Italian copy already locked in §7.2

### 10.2 Update toast (FND-06)

- `/version.json` polling every 5 min while tab visible
- On version mismatch detected → sonner toast `"Nuova versione disponibile"` + body `"Ricarica per aggiornare."` + action button `"Ricarica"`
- Toast has **no auto-dismiss**, only dismisses on action or explicit close
- Action calls `skipWaiting` postMessage → reload
- Pitfall #2 mitigation: never auto-reload (kills user input mid-form)

### 10.3 Safe-area insets

- Top app bar: `padding-top: max(var(--spacing-4), env(safe-area-inset-top))`
- Bottom tab bar: `padding-bottom: max(var(--spacing-4), env(safe-area-inset-bottom))`
- Sheets opening from bottom: `padding-bottom: max(var(--spacing-6), env(safe-area-inset-bottom))`
- Background fills the inset area to avoid two-tone notch effect

### 10.4 Status bar style (manifest + meta)

```html
<meta name="theme-color" content="#FAF9F6" media="(prefers-color-scheme: light)">
<meta name="theme-color" content="#1B1F26" media="(prefers-color-scheme: dark)">
<meta name="apple-mobile-web-app-status-bar-style" content="default">
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-title" content="Wellness Buddy">
```

### 10.5 Sync status indicator (Pitfall #1, Trust signal)

Persistent `SyncStatusPip` top-right of `AppBar`:
- Green dot + "Sincronizzato" tooltip → all good
- Yellow + "In sincronizzazione" + count of pending mutations
- Red + "Sincronizzazione non riuscita" + tap to retry
- Gray + "Offline" + last-synced timestamp

This is non-negotiable per research — it's the trust signal that prevents "this app deleted my data" support tickets when iOS evicts storage.

## 11. Loading / Empty / Error State Patterns

### 11.1 Loading rules

| Surface | Pattern | Why |
|---------|---------|-----|
| `/today` first paint | LoadingSkeleton matching meal-card layout | Avoid spinner-only — tells user the shape of what's coming |
| Weight chart | Skeleton bars at chart shape | Same |
| Form submit | Button shows inline spinner replacing label, button stays width-stable | UX pitfall — never resize button on click |
| Plan parsing | Skeleton + "Sto leggendo il piano..." caption | Set expectation |
| Initial app boot | Logo splash 500ms max, then UI; no spinner unless >1.5s elapsed | Avoid flicker |
| Resync after Dexie wipe | Full-screen state with caption "Sto ricaricando i tuoi dati..." | Pitfall #1 trust signal |

### 11.2 Empty state rules

Every empty state must have:
1. **Storyset illustration** (200×200 mobile / 280×280 desktop), `aria-hidden` if decorative or `<title>`+`aria-labelledby` if meaningful
2. **Heading** (Body emphasis, italic discouraged)
3. **Body copy** explaining what comes next (≤2 sentences)
4. **Primary CTA** unless state is informational-only

Forbidden: empty `<div>` with text only, lorem ipsum spinner, "Nothing here yet."

### 11.3 Error state rules

- **Inline form error:** below field, `role="alert"`, lucide `AlertCircle` 16px icon, `--color-destructive` text, italian copy. Never red border alone.
- **Section-level error:** Replace section content with `ErrorBoundary` component (illustration + heading + body + retry CTA).
- **Toast error:** sonner `error` variant, dismiss 6s, includes recovery hint. Never just "Errore".
- **Page-level error:** Full ErrorBoundary fallback + "Torna a Oggi" escape hatch.

### 11.4 Offline pattern

- Sync pip shows `Offline`
- Mutations queue silently to Dexie outbox
- Toast on first offline action: `"Nessuna connessione. Le modifiche verranno inviate quando torni online."` (one toast, not per-action)
- On reconnect: silent flush + sync pip flips to `Sincronizzato`. No celebration toast (would be noise).
